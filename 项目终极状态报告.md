# 项目终极状态报告

## 1. 项目概览

### 项目基本信息
- **项目名称**：王者战队助手系统
- **技术栈**：
  - Next.js ^14.0.0
  - React ^18.0.0
  - Supabase (Database, Auth, Storage)
  - Tailwind CSS ^3.0.0
  - TypeScript ^5.0.0
  - Chart.js ^4.5.1 (数据可视化)
  - Framer Motion ^12.38.0 (动画)

### 代码库规模
- **总文件数**：146 个（TypeScript/JavaScript 文件）
- **总代码行数**：33,645 行
- **主要功能模块**：8 个（认证、战队管理、智能分组、论坛、交易、AI 助手、数据可视化、用户中心）
- **数据库表**：10+ 个核心业务表
- **API 路由**：17 个
- **前端页面**：33 个

## 2. 架构图（文字描述）

### 前端架构
- **组件化结构**：采用 Next.js 14 App Router，组件化设计，支持 Server Components 和 Client Components
- **布局系统**：使用 `ClientLayout.tsx` 作为全局布局，`PageLayout.tsx` 作为页面容器
- **状态管理**：使用 React Context API（`AuthContext`）管理用户认证状态
- **组件拆分**：已将大型组件拆分为多个子组件（如 Navbar、AI Chat）

### 后端架构
- **Next.js API 路由**：17 个 API 路由，处理数据代理、AI 调用、业务逻辑
- **Supabase 集成**：使用 `@supabase/supabase-js` 进行数据库操作、认证和存储
- **RLS 策略**：为核心业务表启用了 Row Level Security

### 数据流图
```
用户请求 → 前端页面 → API 路由 → Supabase API → 数据库 → 响应 → API 路由 → 前端页面 → 用户
```

## 3. 数据库设计最终版

### 核心业务表

#### 1. teams（战队表）
- **字段**：id (UUID)、name (TEXT)、avatar_url (TEXT)、region (TEXT)、province (TEXT)、city (TEXT)、district (TEXT)、created_at (TIMESTAMP)
- **索引**：无
- **RLS**：已启用，策略：队长可写，所有成员可读

#### 2. team_members（战队成员表）
- **字段**：id (UUID)、team_id (UUID)、user_id (UUID)、role (TEXT)、status (TEXT)、joined_at (TIMESTAMP)
- **索引**：`idx_team_members_team_role` (team_id, role)
- **RLS**：已禁用（避免无限递归）

#### 3. player_profiles（队员资料表）
- **字段**：id (UUID)、user_id (UUID)、team_id (UUID)、game_style (TEXT)、current_status (TEXT)、current_rank (TEXT)、main_positions (JSONB)、rank_updated_at (TIMESTAMP)
- **索引**：`idx_player_profiles_team_user` (team_id, user_id)
- **RLS**：已启用，策略：用户可读写自己的资料，队长可读所有

#### 4. forum_posts（论坛帖子表）
- **字段**：id (UUID)、title (TEXT)、content (TEXT)、author_id (UUID)、team_id (UUID)、category (TEXT)、created_at (TIMESTAMP)、updated_at (TIMESTAMP)
- **索引**：`idx_forum_posts_created_at` (created_at DESC)
- **RLS**：已启用，策略：作者可写，所有人可读

#### 5. team_sales（战队出售表）
- **字段**：id (UUID)、team_id (UUID)、seller_id (UUID)、goods_type (TEXT)、price (DECIMAL)、status (TEXT)、created_at (TIMESTAMP)、updated_at (TIMESTAMP)
- **索引**：`idx_team_sales_status_created` (status, created_at DESC)
- **RLS**：已启用，策略：卖家可写，所有人可读

#### 6. team_recruits（战队招募表）
- **字段**：id (UUID)、team_id (UUID)、title (TEXT)、content (TEXT)、position (TEXT)、rank (TEXT)、members_needed (INTEGER)、image_url (TEXT)、created_at (TIMESTAMP)
- **索引**：无
- **RLS**：未启用

#### 7. group_batches（分组批次表）
- **字段**：id (UUID)、team_id (UUID)、batch_name (TEXT)、total_groups (INTEGER)、created_at (TIMESTAMP)
- **索引**：无
- **RLS**：未启用

#### 8. profiles（用户资料表）
- **字段**：id (UUID)、nickname (TEXT)、avatar (TEXT)、created_at (TIMESTAMP)
- **索引**：无
- **RLS**：未启用

### 表关系
- `teams` ← `team_members` (1:N)
- `teams` ← `player_profiles` (1:N)
- `teams` ← `forum_posts` (1:N)
- `teams` ← `team_sales` (1:N)
- `teams` ← `team_recruits` (1:N)
- `teams` ← `group_batches` (1:N)
- `profiles` ← `team_members` (1:N)
- `profiles` ← `player_profiles` (1:N)
- `profiles` ← `forum_posts` (1:N)
- `profiles` ← `team_sales` (1:N)

## 4. API 清单

### 1. /app/api/ai/chat/route.ts
- **方法**：POST
- **功能**：代理 AI 聊天请求到 Zhipu API
- **分页**：否
- **认证**：不需要
- **示例请求**：`{ "messages": [{ "role": "user", "content": "如何提高战队胜率？" }] }`
- **示例响应**：`{ "choices": [{ "message": { "role": "assistant", "content": "提高战队胜率的方法包括..." } }] }`

### 2. /app/api/recruits/list/route.ts
- **方法**：GET
- **功能**：获取招募列表（支持分页）
- **分页**：是（page, limit）
- **认证**：不需要
- **示例请求**：`/api/recruits/list?page=1&limit=20&teamId=123`
- **示例响应**：`{ "data": [...], "total": 100, "page": 1, "limit": 20, "totalPages": 5 }`

### 3. /app/api/sales/list/route.ts
- **方法**：GET
- **功能**：获取战队出售列表（支持分页）
- **分页**：是（page, limit）
- **认证**：不需要
- **示例请求**：`/api/sales/list?page=1&limit=20&status=active`
- **示例响应**：`{ "data": [...], "total": 50, "page": 1, "limit": 20, "totalPages": 3 }`

### 4. /app/api/forum/posts/list/route.ts
- **方法**：GET
- **功能**：获取论坛帖子列表（支持分页）
- **分页**：是（page, limit）
- **认证**：不需要
- **示例请求**：`/api/forum/posts/list?page=1&limit=20&category=技术讨论`
- **示例响应**：`{ "data": [...], "total": 200, "page": 1, "limit": 20, "totalPages": 10 }`

### 5. 其他 API 路由
- **用户认证**：登录、注册、登出
- **战队管理**：创建、更新、删除战队
- **成员管理**：添加、移除、更新成员
- **智能分组**：创建分组、更新分组
- **数据统计**：获取战队统计数据

## 6. 安全状况

### RLS 策略覆盖
- **teams**：队长可写（user_id = auth.uid() 且 role='队长'），所有成员可读
- **player_profiles**：用户可读写自己的资料，队长可读所有
- **forum_posts**：作者可写，所有人可读
- **team_sales**：卖家可写，所有人可读
- **team_members**：暂时禁用（避免无限递归）

### API 密钥管理
- **ZHIPU_API_KEY**：存储在环境变量中，通过 API 路由代理，无前端暴露
- **SUPABASE_URL** 和 **SUPABASE_ANON_KEY**：存储在环境变量中，前端使用
