## 二、API 路由列表

### 1. /api/applications/route.ts
- **HTTP 方法**：POST
- **功能描述**：处理战队申请的审批和拒绝
- **请求参数/请求体格式**：
  ```json
  {
    "action": "approve" or "reject",
    "applicationId": "UUID",
    "teamId": "UUID",
    "userId": "UUID"
  }
  ```
- **响应格式**：
  ```json
  {
    "success": true,
    "message": "申请已批准" or "申请已拒绝",
    "data": {
      "user_id": "UUID",
      "team_id": "UUID",
      "game_id": "string",
      "current_rank": "string"
    }
  }
  ```

### 2. /api/auth/check-email/route.ts
- **HTTP 方法**：POST
- **功能描述**：检查邮箱是否已注册
- **请求参数/请求体格式**：
  ```json
  {
    "email": "string"
  }
  ```
- **响应格式**：
  ```json
  {
    "exists": boolean
  }
  ```

### 3. /api/group/auto/route.ts
- **HTTP 方法**：POST
- **功能描述**：智能分组功能，根据队员位置和实力自动分组
- **请求参数/请求体格式**：
  ```json
  {
    "team_id": "UUID",
    "locked_members": ["UUID"],
    "mode": "balanced" or "time_priority"
  }
  ```
- **响应格式**：
  ```json
  {
    "success": true,
    "groups": [{
      "id": number,
      "name": "string",
      "members": [{
        "user_id": "UUID",
        "game_id": "string",
        "main_position": "string",
        "second_position": "string",
        "accept_position_adjustment": boolean,
        "available_time": [{}],
        "heroes": ["string"],
        "score": number,
        "recommended_heroes": ["string"],
        "hero_reasons": {}
      }],
      "average_score": number,
      "missing_positions": ["string"],
      "repeated_positions": ["string"],
      "hero_overlap_rate": number,
      "function_coverage": {},
      "role_balance": {},
      "warning": "string",
      "suggested_time": "string",
      "time_fallback_reason": "string"
    }],
    "unassigned": [{}],
    "total_players": number,
    "time_fallback_reason": "string",
    "fallback_reason": "string",
    "tips": ["string"]
  }
  ```

### 4. /api/group/batches/route.ts
- **HTTP 方法**：POST, GET, DELETE
- **功能描述**：管理分组批次
- **请求参数/请求体格式**：
  - POST: `{"team_id": "UUID", "batch_name": "string", "groups": []}`
  - GET: `?team_id=UUID`
  - DELETE: `?batch_id=UUID`
- **响应格式**：
  - POST: `{"success": true, "batch_id": "UUID"}`
  - GET: `{"success": true, "batches": []}`
  - DELETE: `{"success": true}`

- **HTTP 方法**：GET

## 五、已知问题与待修复项

### 1. 智能分组问题
- **问题描述**：当战队只剩下队长一人时，智能分组仍然显示有未分配的队员
- **影响范围**：智能分组功能
- **建议修复方案**：修复数据过滤逻辑，确保只处理有效的队员数据

### 2. 交易系统问题
- **问题描述**：创建商品时出现 "Could not find the 'author_id' column" 错误
- **影响范围**：交易系统
- **建议修复方案**：确保只传递数据库中存在的字段，过滤掉不存在的字段

### 3. 水合错误
- **问题描述**：主页面加载时出现 "Hydration failed" 错误
- **影响范围**：首页
- **建议修复方案**：确保服务器端和客户端渲染一致

### 4. 导航栏遮挡问题
- **问题描述**：多个页面的导航栏遮挡内容区域
- **影响范围**：多个页面
- **建议修复方案**：为所有页面添加适当的顶部内边距

### 5. 数据同步问题
- **问题描述**：浏览器加载时出现老版本页面
- **影响范围**：整个系统
- **建议修复方案**：优化缓存策略，添加版本控制

### 6. 招募卡片问题
- **问题描述**：主页面招募卡片点击查看更多会跳到招募管理页面
- **影响范围**：首页
- **建议修复方案**：修复链接地址，指向正确的招募大厅页面

### 7. 招募卡片闪烁问题
- **问题描述**：跳转到主页面后，招募卡片会闪动加载
- **影响范围**：首页
- **建议修复方案**：优化数据加载逻辑，添加骨架屏

### 8. 卖家信息显示问题
- **问题描述**：战队 ID 出售的卡片下，卖家全部显示：未知用户
- **影响范围**：交易系统
- **建议修复方案**：优化卖家信息获取逻辑

### 9. 移动端菜单位置问题
- **问题描述**：移动端三条杠的菜单弹窗显示位置偏下，不是垂直居中
- **影响范围**：移动端体验
- **建议修复方案**：调整菜单定位样式

### 10. 发帖内容规范问题
- **问题描述**：发帖者的内容没有一个合理的规范，发出的贴吧很杂乱
- **影响范围**：社区系统
