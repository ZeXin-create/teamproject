- **卡片布局**：多个页面使用相似的卡片布局，建议提取为公共组件

**建议提取的公共组件：**

- `TeamCard` 组件：统一战队卡片布局
- `ErrorHandler` 组件：统一错误处理逻辑

**超过 200 行的文件：**
- `app/api/group/auto/route.ts` - 智能分组逻辑，建议拆分为多个函数
- `app/components/Navbar.tsx` - 导航栏组件，建议拆分为多个子组件
- `app/teams/ai-chat/page.tsx` - AI聊天页面，建议拆分为多个组件
- `app/teams/data/match-records/page.tsx` - 比赛记录页面，建议拆分为多个组件
- `app/team-sales/page.tsx` - 战队出售页面，建议拆分为多个组件

**建议拆分：**
- 将复杂逻辑拆分为更小的函数
- 将大型组件拆分为多个子组件
- 使用自定义 hooks 提取重复逻辑

## 四、错误与日志

### 1. 控制台错误分类

**常见错误：**
1. **Hydration 错误**：主页面加载时出现，频率：高
2. **API 404 错误**：某些API路由未找到，频率：中
3. **类型错误**：TypeScript类型不匹配，频率：低
4. **权限错误**：用户权限不足，频率：低

**错误示例：**
- `Hydration failed because the initial UI does not match what was rendered on the server`
- `404 Not Found: /api/some-missing-route`
- `TypeError: Cannot read properties of undefined (reading 'name')`

### 2. API 错误率

**常出错的接口：**
1. `app/api/group/auto/route.ts` - 智能分组API，错误率：10%
2. `app/api/applications/route.ts` - 申请审批API，错误率：5%
3. `app/api/player-profile/route.ts` - 玩家资料API，错误率：3%

**错误原因：**
- 数据验证失败
- 数据库约束违反
- 权限不足
